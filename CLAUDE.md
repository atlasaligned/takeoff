# TAKEOFF — repo conventions

Read `README.md` for architecture. The short version:

- `src/engine/` is pure TS game logic — no React, no DOM, no `Math.random()` (use `state.rng`). All mutations of `GameState` go through engine functions.
- `src/ui/` is a React skin over the engine. UI redesigns must not require engine changes; if you need a derived value, use/add a pure engine helper.
- All tunable numbers live in `src/engine/balance.ts`. After changing balance or systems, run the one harness — `npm run sim` (`src/sim/harness.ts`; add `--fast` for a quick pass, or name sections e.g. `npm run sim -- containment optimal`). Every section plays FULL games (events, gov ladder, jailbreaks — the same world a human plays; events are symmetric, see below), on a fixed seed schedule so every run faces identical worlds. Sections:
  - `containment` — each cheese + 3 reasonable (symmetric, `newTournamentGame`). GATE: a cheese wins ≤5%; exit code 1 otherwise. Cheeses are degenerate SINGLE-IDEA lines (do nothing, just raise, spam one action) — a brain-dead strategy must not win. A sophisticated multi-system playbook that wins is not a gate failure; it's a `strong` strategy (below), not a cheese.
  - `strong` — each `strong` playbook + 3 reasonable (symmetric). NO gate: these are sound multi-system lines (e.g. the rsi-saint endgame) that legitimately win; reported for information only.
  - `fairness` — the 4 reasonable, all seat permutations. Reports each one's share of decided games (watch for one running away).
  - `optimal` — the adaptive `optimal` bot + 3 reasonable. Reports how much the strategy-switcher beats fixed play (chance among 4 seats = 25%).
  - `realgame` — `optimal` in the player seat of the real asymmetric game vs the actual in-game rivals. Closest to what a human faces.
  - Trace one game with `npx tsx src/sim/trace.ts real <strategy> <seed>` or `... sym <s1,s2,s3,s4> <seed>`.
- Events are SYMMETRIC (`save.ts` v9): every lab owns its `govLadder`/`eventCooldowns`/`weeksSinceEvent`, `ActiveEvent.labId` names the target, and every `EventDef`/`ChoiceDef` callback takes `(state, lab, data)`. `tick.ts` rolls events for every alive lab; the human's queue as blocking modals, AI labs auto-answer via `aiEventChoice` (strategy `events` map → `EventDef.aiChoice` → first choice). Event `scope`: `'lab'` (default, per-lab), `'world'` (rolled once by the "world roller" — the player, or first alive lab in sim — so market/adoption shocks don't 4×), `'player-only'` (EU flavor). Board-removal / nationalization end the game for the human but only kill an AI lab.
- Strategies live in `src/engine/strategy.ts` (not in src/sim) because they double as in-game rival personalities: each `LAB_SEEDS` entry names a strategy and `strategyAct()` drives rivals with it (base `rivalAct` housekeeping + the strategy's `extra`, minus its `reserved` research / `ownRuns` training / raises). `runStrategy()` is the shared executor. Strategies fall in three tiers by flag: `cheese: true` (simple exploits, gated ≤5%), `strong: true` (sound multi-system playbooks, reported ungated), and neither (`REASONABLE` — the 4 anchors that fill the symmetric table). `optimal` (`meta: true`, `src/engine/arbiter.ts`) is none of these — it clones the live game, rolls each reasonable playbook to the end over K reseeds every `ARBITER.evalInterval` weeks, and adopts the winner. Harness-validated strategies ARE the rival AI — tune them and the game's opposition changes.
- Design note: RSI (self-growth) is mathematically the *only* route to capability 100 — training FLOP alone can't reach it (the log curve). RSI is deliberately slow and tapers near the ceiling (`RSI_DECEL_CAP`), so the skillful racer line is to grind alignment to the Provable ceiling first, then take RSI last to tip a well-aligned flagship over 100. Nerfing RSI's *reach* (vs its speed) removes racing as a win path entirely.
- Verify before finishing: `npm run lint && npm run typecheck && npm test`. The UI smoke tests (`src/ui/App.test.tsx`) SSR-render every tab against a real mid-game state — keep them passing when changing the UI.
- `GameState` is plain JSON (saves = `JSON.stringify`). Don't put functions, class instances, or `Date` objects in it. Bump `version` in `save.ts` on breaking shape changes.
